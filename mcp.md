# MCP 与工具组合

## 现实校准
**大部分官方源没有 API/MCP**（CNINFO、交易所、股转、政府网、互动易都只能 fetch/抓取）。所以 MCP 层很薄，别指望它挑大梁。工具层的真实构成是：

> **一个企查查 MCP + 一份源路由文档（`sources/official-channels.md`）**

`sources/official-channels.md` 承担的其实是"人肉 API 文档"的角色——告诉执行者每个平台怎么定位到目标文件。

## 值得接的 MCP
| MCP | 用途 | 替代了什么 |
|---|---|---|
| 企查查 MCP | 工商信息、股权穿透、风险信息 | gsxt 的滑块验证码反爬 |

天然契合"审计工具"定位：只读、来源可溯、合规。

## 官方源的处理方式
统一走 **fetch + 源路由文档里的检索路径说明**。批量离线采集为主，不做实时爬取（合规 + 反爬友好）。遵守各站 robots.txt 与合理频率限制。

## 明确不接
- ❌ 荣大二郎神等付费源的接口（不进零预算核心，见 sources D 层）
- ❌ 任何需要绕过登录墙/验证码的自动化（合规红线）
