## 三、Claude Code 配置修改

### 修改文件

`~/.claude/settings.json` (即 `C:\Users\86195\.claude\settings.json`)

### 修改内容

**在 `"effortLevel": "high"` 之后，添加 `"env"` 字段：**

```diff
 {
   "enabledPlugins": {
     "playwright@claude-plugins-official": true
   },
   "autoUpdatesChannel": "latest",
   "theme": "dark",
-  "effortLevel": "high"
+  "effortLevel": "high",
+  "env": {
+    "GITHUB_PERSONAL_ACCESS_TOKEN": "<你的GitHub PAT>"
+  }
 }
```

### 修改目的

让 GitHub MCP 服务器能够读取 `GITHUB_PERSONAL_ACCESS_TOKEN` 环境变量，从而正常调用 GitHub API。

### 关联 MCP 配置

`.claude/plugins/marketplaces/claude-plugins-official/external_plugins/github/.mcp.json` 中通过 `${GITHUB_PERSONAL_ACCESS_TOKEN}` 引用此环境变量，无需额外修改。

## 四、Token 过期后的操作

`claude-code-github-mcp` token 过期后，只需：

1. 在 GitHub 生成新的 fine-grained token（Administration + Contents + Metadata 权限）
2. 修改 `~/.claude/settings.json` 中 `env.GITHUB_PERSONAL_ACCESS_TOKEN` 的值为新 token
