# Normalization 参考 · Claim Schema 与口径规整

**这是所有 playbook 共享的地基。** 每条断言在核验前，必须先规整成本文件定义的标准形态。跳过这一步直接检索，各种口语表达会导致匹配失败——这是事实核验系统最常见的死法。

---

## 一、Claim 五元组 Schema

Step 2 拆出的每条 ①硬事实 / ②预测类断言，规整成以下 JSON 再进入路由：

```json
{
  "claim_id": 1,
  "claim_type": "financial | equity | capacity | industry | policy",
  "metric": "REVENUE",
  "entity": "宁德时代",
  "entity_code": "300750",
  "period": "FY2024",
  "unit": "亿元",
  "value": 382.4,
  "original_text": "去年营收超380亿",
  "derivation": "direct | calculated"
}
```

字段规则：

| 字段 | 规则 |
|---|---|
| `metric` | 必须映射到下方《指标映射表》的标准代号。映射不到 → 标 ⚠️无法规整，不硬猜 |
| `entity` | 公司全称或通用简称 + 股票代码。文章只写"某锂电龙头"→ 先据上下文锁定主体，锁不定则整条 claim 标 ⚠️主体不明 |
| `period` | 标准化为 `FY2024` / `2024Q3` / `2024H1` / `2024Q1-Q3`(累计)。**文章没写期间 → 见下方《期间推断规则》，禁止默认"最新一期"** |
| `unit` | 统一到 亿元 / 万元 / % / pct。比对前先换算到同一单位 |
| `value` | 数值。区间表述（"约30亿""超380亿"）保留修饰词进 `original_text`，value 取中值/边界值并在比对时放宽容差 |
| `derivation` | `direct` = 原文直接给数；`calculated` = 需从披露科目计算（ROE、增速等） |

**这个 schema 同时解决四个问题**：执行规范、版本控制（period 字段）、路由输入统一、后续摸底数据可统计。摸底采集的每条数据都用这个格式存。

## 二、指标映射表（口语 → A股准则科目，封闭集合）

A 股法定披露科目被会计准则强制标准化，同义词空间**有限且封闭**——本表可穷举，遇到表外新词再增补，不做开放式语义匹配。

### 利润表类
| 标准代号 | 准则科目名 | 荐股文口语/别名 |
|---|---|---|
| `REVENUE` | 营业收入 | 营收、收入、销售额、Revenue、总收入 |
| `NET_PROFIT` | 净利润 | 纯利、盈利、赚了X亿 |
| `NET_PROFIT_ATTR` | 归属于母公司股东的净利润 | 归母净利、归母净利润 |
| `NET_PROFIT_DEDUCT` | 扣除非经常性损益的净利润 | 扣非净利、扣非 |
| `GROSS_MARGIN` | 毛利率 | 毛利、毛利水平 |
| `NET_MARGIN` | 净利率 | 净利水平 |
| `OP_PROFIT` | 营业利润 | — |
| `EPS` | 基本每股收益 | 每股赚、EPS |

⚠️ **净利润三兄弟是荐股文最爱混淆的陷阱**：文章说"净利润 10 亿"，年报可能是归母 8 亿、扣非 6 亿。规整时若原文未指明，`metric` 记 `NET_PROFIT`，比对时**三个口径都查**，报告中注明"原文未指明口径，按 X 口径为✅/❌"。

### 资产负债表类
| 标准代号 | 准则科目名 | 口语/别名 |
|---|---|---|
| `TOTAL_ASSETS` | 总资产 | 资产规模 |
| `NET_ASSETS` | 归属于母公司股东的净资产 | 净资产、账面价值 |
| `DEBT_RATIO` | 资产负债率 | 负债率、杠杆 |
| `GOODWILL` | 商誉 | — |
| `CASH` | 货币资金 | 账上现金、现金储备 |

### 现金流类
| 标准代号 | 准则科目名 | 口语/别名 |
|---|---|---|
| `OCF` | 经营活动产生的现金流量净额 | 经营现金流、造血能力 |
| `CAPEX` | 购建固定资产、无形资产和其他长期资产支付的现金 | 资本开支 |

### 派生指标（derivation = calculated，公式写死）
| 标准代号 | 公式 | 口语/别名 |
|---|---|---|
| `ROE` | 归母净利润 ÷ 期末归母净资产（若原文明示加权则用加权） | 净资产收益率 |
| `ROA` | 净利润 ÷ 期末总资产 | 总资产收益率 |
| `YOY_GROWTH` | (本期−上年同期) ÷ 上年同期 | 同比增长、增速、涨了X% |
| `GROSS_MARGIN_CHG` | 本期毛利率 − 上期毛利率（结果单位=pct） | 毛利率提升X个点 |

### 经营类（非报表科目，源为公告/招股书）
| 标准代号 | 说明 | 口语/别名 |
|---|---|---|
| `ORDER_BACKLOG` | 在手订单（**注意：非强制披露项，查无此据高发区**） | 在手订单、订单额 |
| `CAPACITY` | 产能 | 产能、年产X |
| `MARKET_SHARE` | 市占率（必须带口径，见 playbook 04） | 市场份额、行业第一 |

## 三、期间推断规则

文章不写期间是常态，禁止默认"最新一期"，按此顺序推断：

1. **上下文锚定**：文中出现"去年/2024年报/三季报"等 → 直接锁定。
2. **发文时间锚定**：有发文日期 → "去年" = 发文年份−1 的 FY；"最新季报" = 发文日之前最近的已披露季度。
3. **数值反查**（前两步都失败时）：拿 value 去近三期披露里反查——只在某一期对得上 → 推断为该期并在报告注明"期间为反查推断"；多期都对不上 → ⚠️查无此据；多期都能对上 → 取最近一期。
4. 全部失败 → 该 claim 标 ⚠️期间不明，报告中如实说明，不硬判。

## 四、口径对齐规则（比对前的强制检查）

比对前逐项过一遍，**口径不同不算矛盾**，先对齐再判：

| 检查项 | 规则 |
|---|---|
| 单季 vs 累计 | Q3 单季 ≠ Q1-Q3 累计。文章"三季度营收" 默认歧义，两个口径都查 |
| 合并 vs 母公司 | 一律默认合并报表口径；文章明示母公司才切换 |
| 归母 vs 全口径 vs 扣非 | 见净利润三兄弟规则 |
| % vs pct | "毛利率提升5%"（相对）≠"提升5个百分点"（绝对）。歧义时两种都算 |
| 单位换算 | 万元/亿元/百万统一到亿元再比 |
| 追溯调整 | 年报重述前期数据时，以**最新披露版本**为基准，差异处注明"经追溯调整" |

## 五、数值容差规则

| 原文表述 | 判定容差 |
|---|---|
| 精确数（"382.4亿"） | ±0.5%（容四舍五入）内 = ✅ |
| 约数（"约380亿""近400亿"） | ±5% 内 = ✅ |
| 边界词（"超/破/不到 X"） | 按不等号方向判定 |
| 比率类（毛利率38%） | ±0.5 pct 内 = ✅ |

超出容差 → ❌矛盾，报告给出双方数字；披露中该科目/期间不存在 → ⚠️查无此据。

## 边界声明

本表覆盖 A 股准则口径。**不做**开放式语义匹配、不预防 Adjusted/Core/Recurring Revenue 这类 non-GAAP 表述（那是美股语料，A 股荐股文中不存在该问题；若未来扩市场再增补）。遇到映射不到的指标，标 ⚠️无法规整并原文记录——这些记录正是摸底时扩充本表的原料。
