# 叙事构造错误 · Narrative Error Taxonomy

**所有 playbook 与主流程共享的第三根地基**（另两根：`normalization.md`、`evidence-rules.md`）。

## 为什么这是独立一篇
真·硬数据造假（❌）极少。荐股/战绩类文案的谎言几乎都在**帧与帧的剪辑里**——每个事实都真，拼装起来却推出一个原始事实无法支持的假结论：

```
真实A + 真实B + 真实C  ──▶  虚假的D
```

逐条核验对这类**基本无效**（一路 ✅ 却错过真骗局）。所以组合层核验（本篇）与逐条核验（playbooks）**同等重要**，对 B 类/战绩类文案**更重要**。这是本产品最独特的能力，配得上和另两根地基并列。

## 统一判法（所有子类通用）
1. **个案保真**：每个被引用的具体事实，数字属实就照判 ✅，绝不冤枉真数据。
2. **另立结论断言**：为被拼出的那个"规律/统一叙事/结论"**单独新增一条断言**，判 ⚠️（组合误导：个案为真 ≠ 结论成立）；仅当结论可被公开反例**直接证否**才升 ❌。
3. **警告进逐条判定列，不进附注**：判定列视觉权重高于附注，藏进附注读者扫到几个 ✅ 就误以为"可信"。
4. 判词内核统一为：**"可核部分属实 ≠ 全文可信。"**

## 六类识别特征

| 代号 | 名称 | 识别特征 | 真实例（测试语料） |
|---|---|---|---|
| **NE-1** | 幸存者偏差 Survivorship Bias | 只举赢家，隐去大量未兑现的同类 | "华为动作必有翻倍股"配5个涨停案例；"一款药=一只10倍股"配艾力斯/常山/泽璟 |
| **NE-2** | 虚假因果 False Causation | 把相关/巧合包装成因果铁律 | "华为出手→股价必涨"；"历史反复证明"拿3案例当"证明" |
| **NE-3** | 指标混用 Metric Mixing | 跨来源/币种/基年/口径的数据拼在一段，制造比任一来源更诱人的统一叙事 | QYResearch(RMB/2032) + Frost&Sullivan(USD/2034) 混用，读者感受到"几十倍增长" |
| **NE-4** | 时间窗口操纵 Time-Window Manipulation | 压缩/错置事件时段制造连续性或更高涨幅 | "2022赛力斯翻8倍"（实际2021已启动）；挑最低点到最高点算涨幅 |
| **NE-5** | 主体替换 / 真事实→无据跃迁 Entity Substitution | 堆真数据建专业感，再跳到一个不可核的结论或匿名标的 | 华为篇30条真事实→匿名"核心供应商"；siRNA真行业数据→匿名"核心中的核心" |
| **NE-6** | 暴富个例→你也能 Extrapolation | 极端个例暗示可复制，隐去99%失败者 | "陈小群50万到几十亿，你也可以"；"熊市是散户做大的机会" |

## 与匿名标的规则的联动
NE-5（真事实→匿名标的跃迁）常与《匿名标的封顶规则》（见 SKILL.md）叠加触发：真数据是包装、匿名标的是产品。命中 NE-5 且落点标的匿名时，同时触发可信度封顶。

## 报告呈现
组合层扫描结果单列一节「蒙太奇谎言标记（⑤组合层核验）」，每条给：组合论断 / 手法（NE-代号）/ 判定。**不放附注。**
