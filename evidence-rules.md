# 证据优先级与冲突消解规则

所有 playbook 共享。回答两个问题：多个来源都有这个数字时**先查谁**；来源之间**打架时信谁**。

---

## 一、证据优先级（检索顺序 = 采信顺序）

| 级别 | 来源 | 说明 |
|---|---|---|
| **P1** | 年度报告（经审计） | 最高基准。经审计的数字是唯一"硬到底"的证据 |
| **P2** | 半年报 / 季度报告 | 未经审计但属法定披露 |
| **P3** | 临时公告（业绩快报/预告、权益变动书、投资者关系活动记录表） | 业绩快报晚于年报时以年报为准 |
| **P4** | 招股书 / 股转说明书 / 募集说明书 | 权威但有时点性——上市越久越可能过期 |
| **P5** | 互动易 / 上证e互动答复、业绩说明会实录 | 公司口径但非法定披露，交易所规则明确"以指定媒体披露为准" |
| **P6** | 公司官网 / 官方公众号 | 官方渠道但无信披效力 |
| **P7** | 第三方（东财/同花顺/雪球/企查查等） | **仅作线索与定位，永远不作判真伪基准**（铁律二） |

检索时从高到低找，**在 P1-P4 找到明确答案即停**，不必穷尽所有层级。P5-P6 只在高层级完全无此信息时才作为"公司曾如此表述"的记录（此时判 ⚠️查无此据 + 附注"公司在互动平台曾称…"，而非判✅）。

## 二、冲突消解规则

1. **层级不同 → 取高层级**。年报 vs 互动易答复打架，信年报，报告注明低层级来源的矛盾表述（这本身是有价值的发现）。
2. **同层级、不同时点 → 取最新披露**（含追溯调整/更正公告；更正公告视同其所更正文件的层级）。
3. **同层级、同时点仍矛盾**（如年报正文与附注不一致）→ 不硬判，报告如实呈现两处数字并标 ⚠️披露自身存在不一致——这是比✅/❌更重要的发现。

## 三、证据出身标注（写在来源列，不单列、不评分）

证据的出身在报告**来源列**如实注明即可（不单列证据强度列、不评数值分——那与来源冗余，且 0.x 分会注入伪精度）。全局可信度由报告顶部的四档总评承载。出身类别：

| 标签 | 含义 | 对应 |
|---|---|---|
| `[直接披露]` | 法定披露文件中直接列示的数字 | P1-P4 直接命中 |
| `[计算派生]` | 由披露科目按公式计算（ROE、增速等），报告须写明公式与取数 | P1-P4 + derivation=calculated |
| `[行情数据]` | 来自官方行情流（股价/市值/PE/PB/分位），权威出处为交易所行情 | 行情流（playbook 07） |
| `[公司表述]` | 来自互动答复/说明会/官网的公司口径，非法定披露 | P5-P6 |
| `[无证据]` | 各层级均无此信息 | ⚠️查无此据 |

`[直接披露]` ≈ `[行情数据]` > `[计算派生]` > `[公司表述]`（前两者分属披露流与行情流，各自权威）。当一条✅仅由 `[公司表述]` 支撑时，报告必须注明"该信息未见于法定披露文件"。

## 四、来源溯源规则（每条命中都要能回到源头）

核验命中的每条来源，报告都要让使用者能一键回到原文自查。呈现分三态：

| 状态 | 写法 | 用于 |
|---|---|---|
| **可溯源** | 贴**真实可点击的原始 URL** | 实际抓取/访问过的页面（公告详情页、行情页、新闻原文等） |
| **待溯源** | `[待溯源：源 + 定位]`，如"CNINFO · 紫金 2025年报 · 第X节"或"AKShare `stock_a_indicator_lg`" | 明确知道去哪查、但当前环境未实际抓取（无接口/无终端时） |
| **无来源** | `—` | 查无此据、个人数据、纯观点 |

**防伪造红线（最重要）**：**只贴真正抓取或访问过的 URL，绝不根据记忆、规律或公告编号规则拼造链接。** 溯源功能最危险的失败模式，就是模型编一个"看起来像真的"的假链接（如猜一个 CNINFO 公告号）——那比不给链接更坏，它给使用者虚假的确定感。没抓到就用"待溯源"如实标注检索路径，把复核交给使用者，**永远不猜链接**。
